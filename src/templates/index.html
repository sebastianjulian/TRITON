TRITON

Autonomous Submarine Navigation System

An advanced dual-platform submarine navigation system that evolved from our CanSat project. TRITON combines sophisticated sensor technology with real-time data transmission to provide comprehensive underwater monitoring and navigation capabilities for autonomous submarine operations.

Project Overview

TRITON represents the evolution of our satellite simulation project into a sophisticated autonomous submarine navigation system. Designed for real-world underwater operations, TRITON integrates environmental monitoring with inertial navigation to provide comprehensive submarine guidance and mission-critical data collection capabilities.

Environmental Monitoring

Advanced BME280 sensor provides precise temperature, humidity, pressure, and altitude measurements. Pressure readings enable accurate depth calculation for underwater navigation, while environmental data supports mission planning and safety protocols.

Inertial Navigation

MPU6050 inertial measurement unit delivers 6-axis motion sensing with 3-axis accelerometer and 3-axis gyroscope. This provides real-time orientation tracking, motion detection, and navigation support for autonomous submarine operations.

Wireless Communication

LoRa long-range wireless technology enables reliable data transmission between submarine and surface operations. Real-time sensor data streaming supports remote monitoring and mission control from considerable distances.

Real-time Dashboard

Web-based mission control interface provides live visualization of all sensor data, statistical analysis, and comprehensive logging. Chart.js integration delivers professional-grade data visualization for operational analysis.

Data Logging

Comprehensive CSV logging with automatic archiving ensures mission data integrity. Threshold-based data transmission reduces network overhead while maintaining critical data resolution for post-mission analysis.

Autonomous Operation

Raspberry Pi-based sensor collection with intelligent error handling and graceful degradation. System continues operation even with partial sensor failures, ensuring mission continuity and data collection reliability.

Python API Reference

Sensor Collection Module (test.py)

Key Functions:
  • read_bme280() - Read environmental sensor data
  • read_mpu6050() - Read inertial measurement data
  • send_lora_data(data) - Transmit data via LoRa
  • update_web_dashboard(data) - Send data to web server
  • should_transmit(current, previous) - Check transmission thresholds
Data Thresholds:
  • Temperature BME280: 0.25°C
  • Humidity: 1.0%
  • Pressure: 0.5 hPa
  • Altitude: 0.5m
  • Acceleration (all axes): 0.25 m/s²
  • Gyroscope (all axes): 5.0°/s
  • Temperature MPU: 0.25°C

Data Reception Module (lorareceivertest.py)

Key Functions:
  • receive_lora_data() - Receive LoRa transmissions
  • parse_sensor_data(raw_data) - Parse received data
  • log_to_csv(data) - Write data to CSV file
  • calculate_statistics(data_history) - Compute min/max/avg
  • archive_old_logs() - Manage log file rotation

Web Server Module (app.py)

Key Functions:
  • update_data_history(new_data) - Add data to history buffer
  • generate_chart_data() - Format data for Chart.js
  • serve_log_files() - Handle file downloads
  • cleanup_port(port) - Port management utility

Frequently Asked Questions

How do I set up the Raspberry Pi for TRITON?

Install Raspberry Pi OS Lite, enable I2C via raspi-config, install Python dependencies: pip install adafruit-circuitpython-bme280 mpu6050-raspberrypi pyserial requests. Connect sensors according to the wiring diagrams and run python src/test.py.

What is the maximum transmission range?

LoRa modules can achieve up to 15km line-of-sight transmission range in optimal conditions. Underwater operations will significantly reduce this range to approximately 50-100 meters depending on water conditions, depth, and antenna positioning.

How long does the battery last?

With a 2200mAh 7.4V Li-Po battery, TRITON operates for approximately 4 hours of continuous data collection and transmission. Power consumption can be optimized by adjusting transmission intervals and sensor reading frequency.

Can I modify the sensor thresholds?

Yes, transmission thresholds are configurable in src/test.py. Modify the threshold values in the should_transmit() function. Lower thresholds increase data frequency but consume more power and bandwidth.

How do I calibrate the sensors?

BME280: Adjust sea level pressure in the code (currently 993.9 hPa) based on your location. MPU6050: Run calibration routine by placing the device on a level surface and recording offset values for accelerometer and gyroscope.

What happens if sensors fail?

TRITON includes comprehensive error handling. If a sensor fails, the system continues operating with remaining sensors and logs error messages. The web dashboard displays "Sensor Error" for failed components while maintaining data collection from functional sensors.

How do I access the web dashboard remotely?

The Flask server runs on 0.0.0.0:5000 and is accessible from any device on the same network. Use the PC's IP address: http://[PC_IP]:5000. For internet access, configure port forwarding on your router or use VPN.

Can TRITON work without LoRa communication?

Yes, TRITON can operate in standalone mode. Data is logged locally on the Raspberry Pi's SD card. The web dashboard can still function using test data generation, and logs can be retrieved manually after the mission.

How do I troubleshoot I2C sensor issues?

Use i2cdetect -y 1 to scan for connected devices. BME280 should appear at 0x76 and MPU6050 at 0x68. Check wiring connections, ensure 3.3V power supply, and verify pull-up resistors on SDA/SCL lines. Enable I2C in raspi-config if not already enabled.

What file formats are supported for data export?

Currently, TRITON exports data in CSV format with MET timestamps. Files include sensor readings, elapsed time, and statistical summaries. CSV files can be opened in Excel, MATLAB, Python pandas, or any data analysis software for further processing.

Mission Capabilities

TRITON is engineered for comprehensive submarine operations with advanced monitoring and navigation capabilities. The system provides continuous environmental monitoring for depth calculation through pressure readings, inertial navigation support through accelerometer and gyroscope data, and real-time wireless data transmission for surface monitoring and control.

The dual-platform architecture ensures operational redundancy and reliability. The Raspberry Pi handles autonomous sensor collection and LoRa transmission, while the PC base station manages data reception, logging, statistical analysis, and web-based mission control interface. This design enables real-time monitoring of submarine operations from considerable distances while maintaining comprehensive data logging for post-mission analysis.
